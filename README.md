# sql_backup
## Task 1
## 1.1
## Для восстановления данных за предыдущий день наиболее подходящим вариантом будет полное ежедневное резервное копирование (full backup). ## Этот метод создает полную копию базы данных каждый день, что позволяет восстановить состояние базы на конец предыдущего дня. 
## Преимущества:
## - простота восстановления; 
## - полная сохранность данных;
## - минимальное время на восстановление конкретной точки.
## 1.2
## Для восстановления данных за час до поломки рекомендуется использовать комбинацию:
## - ежедневное полное резервное копирование;
## - часовые инкрементальные бэкапы; 
## - ведение транзакционных логов (WAL в PostgreSQL)
